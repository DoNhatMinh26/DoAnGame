QUAN TRỌNG:
-----------
Scene này CHỈ CÓ MULTIPLAYER, không có single-player!
→ Dùng MultiplayerDragAndDrop (KHÔNG phải DragAndDrop)
→ KHÔNG cần MultiplayerDragDropAdapter


CÁCH SETUP NHANH NHẤT:
=======================

BƯỚC 1: Chạy Auto-Setup Tool
-----------------------------
Menu Unity: Tools → Multiplayer Battle → Auto-Setup References

Tool sẽ tự động:
✅ Thêm MultiplayerDragAndDrop vào Answer_0/1/2/3
✅ Thêm CanvasGroup và Image component (nếu thiếu)
✅ Gán myText reference tự động
✅ Gán tất cả references vào UIMultiplayerBattleController
✅ Gán references cho MultiplayerHealthUI


BƯỚC 2: Save Scene
-------------------
Ctrl+S để lưu thay đổi


BƯỚC 3: Validate
-----------------
Menu: Tools → Multiplayer Battle → Validate Setup

Hoặc trong game bấm F3


BƯỚC 4: Test
------------
1. Chạy Host
2. Thử kéo thả đáp án
3. Kiểm tra console log:
   - "[MultiplayerDragAndDrop] Begin drag: 14"
   - "[MultiplayerDragAndDrop] End drag: 14, dropped on: Slot"
   - "[MultiplayerDragAndDrop] Answer submitted: 14"

TROUBLESHOOTING:
================

VẤN ĐỀ 1: Không gán được Answer vào Answer Choices
---------------------------------------------------
NGUYÊN NHÂN: Answer chưa có MultiplayerDragAndDrop component
GIẢI PHÁP: Chạy Auto-Setup tool


VẤN ĐỀ 2: Kéo được nhưng không submit
--------------------------------------
NGUYÊN NHÂN: 
- Slot không có tag "Slot"
- BattleController chưa gán
- EventSystem không active

GIẢI PHÁP:
- Kiểm tra Slot tag
- Chạy Validate tool
- Kiểm tra EventSystem trong scene


VẤN ĐỀ 3: Console log "BattleController is NULL"
-------------------------------------------------
NGUYÊN NHÂN: UIMultiplayerBattleController không tìm thấy
GIẢI PHÁP: 
- Kiểm tra GameplayPanel có component UIMultiplayerBattleController
- Chạy Auto-Setup tool


VẤN ĐỀ 4: Timer không chạy
---------------------------
NGUYÊN NHÂN: TimerFillImage chưa gán
GIẢI PHÁP:
- Chạy Auto-Setup tool
- Hoặc gán thủ công: HealthBarContainer → Timer Fill Image → TimerFill
