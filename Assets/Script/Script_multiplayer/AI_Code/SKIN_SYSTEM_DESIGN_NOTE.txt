THIET KE CHUAN CHO CASE NAY

Thiet ke chuan cho case cua ban la:
Logic co dinh, giao dien thay bang Skin System.

1) Tong quan kien truc
- Game Logic Layer:
  + Sinh cau hoi, cham diem, tang do kho, multiplayer.
  + Khong chua thong tin mau/sprite/font.
- Presentation Layer:
  + Hien thi du lieu len UI (timer, hp, diem, cau hoi, dap an).
- Theme/Skin Layer:
  + Quy dinh hinh anh, mau sac, font, icon, vfx, am thanh.

2) Du lieu can co
- ThemeConfig (ScriptableObject):
  + themeId, themeName
  + backgroundSprite, panelSprite, buttonSprite
  + primaryColor, accentColor, dangerColor
  + tmpFontAsset
  + correctVfx, wrongVfx
  + bgm, sfxPack

- LevelVisualProfile (ScriptableObject hoac JSON):
  + levelFrom, levelTo
  + themeId
  + variantPool (A/B/C)
  + transitionType

- PlayerVisualProgress:
  + theme dang dung
  + variant da dung gan nhat
  + seed random de tranh lap lai nha chan

3) Code design khuyen nghi
- ThemeManager:
  + ApplyTheme(themeId)
  + ApplyVariant(levelId)
  + NotifyThemeChanged()

- IThemeTarget:
  + ApplyTheme(ThemeConfig theme)

- UIThemeBinder:
  + Bind danh sach Image, TMP_Text, Button, AudioSource
  + Nghe su kien doi theme va cap nhat tu dong

4) Quy trinh van hanh
- Khong hardcode mau/sprite trong script gameplay.
- Khong clone scene cho tung level de doi giao dien.
- Moi thay doi visual di qua ThemeManager.
- Logic level van giu nguyen, skin thay doi theo level/chuong.

5) Loi ich
- De mo rong nhieu level ma khong doi logic.
- De can bang va A/B test.
- De bao tri do code gameplay va UI duoc tach ro.
- Giam nha chan vi cung mot logic nhung giao dien duoc bien doi.
