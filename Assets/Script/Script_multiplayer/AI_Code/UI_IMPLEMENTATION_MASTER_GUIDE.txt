================================================================
 HƯỚNG DẪN TRIỂN KHAI UI DOANGAME (UI 1 → UI 6 + FIREBASE)
 Bản chụp: 2026-04-02
================================================================

Tài liệu này phản ánh **các scene hiện có**:
- Scene **WELCOME SCREEN** → UI 1 (Welcome Screen / Intro)
- Scene **GameUIPlay** → UI 2–UI 6 (Auth, Login, Register, Main Menu, Mode)
- Scene **GameUIPlay** → UI 15-16 (Multiplayer Room + Battle)
  Mọi script nằm trong `Assets/Script/Script_multiplayer/AI_Code/CODE`.

Khi tạo các UI mới (6 → 14) hãy mở rộng file theo đúng khung này.

🔥 **VỀ FIREBASE**: Dự án đã có Firebase Auth + Realtime Database setup
- Xem chi tiết: `FIREBASE_STORAGE_SOLUTION.md`
- Dữ liệu được lưu: Avatar, Tên, Điểm, Level, Rank, Lịch sử trận, Cài đặt
- Cross-device: Đăng nhập máy khác vẫn có toàn bộ data

----------------------------------------------------------------
 TỔNG QUAN FLOW
----------------------------------------------------------------
```
Scene: WELCOME SCREEN
└─ UI1 Welcome Screen
  ↳ Play → (load GameUIPlay) → UI2 Welcome Panel

Scene: GameUIPlay
UI2 Welcome Panel ─┬─ Chơi Nhanh → UI5 Main Menu
          ├─ Đăng Nhập → UI3 Login
          ├─ Đăng Ký → UI4 Register
          └─ Settings → (UI Settings sẽ bổ sung sau)

UI3 Login ─┬─ Back → UI2
     ├─ Đăng Nhập thành công → UI5
     └─ Đăng Ký → UI4

UI4 Register ─┬─ Back → UI2
        └─ Hoàn tất thành công → UI3

UI5 Main Menu (Hub) ─┬─ Chơi → UI6 Mode Selection
    ├─ BXH / Hồ sơ / Settings → placeholder
    └─ Logout → UI2

UI6 Mode Selection ─┬─ Solo → UI7 Level Selection
      ├─ Multiplayer → UI7 Level Selection
      └─ Back → UI5

UI15 Multiplayer Room ─┬─ Tạo phòng (Host) → hiện mã phòng
     ├─ Vào nhanh (Quick Join) → vào phòng có sẵn
     ├─ Nhập mã phòng → Join theo code
         └─ Đủ 2 người + Host nhấn Bắt đầu → UI16

UI16 Multiplayer Battle ─┬─ Player 1 (local)
       └─ Player 2 (đối thủ/AI fallback)
```

 QUẢN LÝ FLOW (GameUIPlay)
----------------------------------------------------------------
Thành phần: `UIFlowManager` đặt trên `UIFlowManagerRoot`
- `Màn hình bắt đầu` = `WelcomeIntro` (nếu chạy 1 scene có UI1) hoặc `WelcomeAuth` (nếu chạy scene GameUIPlay riêng)
- `Panels mapping`:
  1. `WelcomeIntro` → `WELCOMESCREEN (UIWelcomeIntroController)` khi chạy 1 scene
     (nếu tách 2 scene thì không map UI1 trong GameUIPlay)
  2. `Login` → `LoginPanel (UILoginPanelController)`
  3. `Register` → `RegisterPanel (UIRegisterPanelController)`
  4. `MainMenu` → `MainMenuPanel (UIMainMenuController)`
  5. `ModeSelection` → `ModSelectionPanel (1) (UIModeSelectionController)`
- Khi có panel mới, bổ sung entry tương ứng.

Lưu ý:
- Không thêm event trong `Button > On Click()`. Các controller đã đăng ký
  listener ở `Awake()`.
- Mọi Button/Input/TMP_Text phải được kéo vào serialized field tương ứng trong
  Inspector; nếu thiếu Unity sẽ highlight đỏ.

----------------------------------------------------------------
 NAVIGATION CONFIG (Drag & Drop)
----------------------------------------------------------------
- Các controller điều hướng (kế thừa `FlowPanelController`) có thêm mục
  `Navigation Configs` trong Inspector.
- Mỗi panel (trừ UI1 nếu không cần quay lại) có thể gán nhanh nút Back tại
  field `backButton` trong mục `Common Buttons (Nút dùng chung)`.
  Khi gán field này, bấm Back sẽ tự gọi `UIFlowManager.Back()`.
- Mỗi entry = 1 button:
  * `Button`: nút cần điều hướng.
  * `Action`: `ShowScreen`, `Back`, `ShowSettings`, `ReturnFromSettings`.
  * `Target Screen`: đích tới khi action = `ShowScreen`.
  * `Origin Screen`: màn hình gốc dùng cho Settings để biết quay lại đâu.
  * `Override Push History`: bật nếu muốn tự quyết pushHistory (ví dụ Logout).
- Ví dụ:
  * Welcome Screen: `Play` → action `ShowScreen`, target `WelcomeAuth`.
  * Main Menu Settings: action `ShowSettings`, origin `MainMenu`.
  * Settings Close: action `ReturnFromSettings`.
- Lợi ích: thay đổi tuyến UI chỉ cần chỉnh Inspector, không phải sửa code.

 HƯỚNG DẪN 5 BƯỚC (VN)
 1. Chọn panel cần cấu hình (ví dụ `GameUICanvas/MainMenuPanel`).
 2. Trong Inspector kéo xuống mục `Navigation Configs`, nhấn dấu `+` để tạo dòng mới.
 3. Kéo thả button từ hierarchy vào ô `Button`, sau đó chọn `Action`.
 4. Nếu `Action = ShowScreen` thì chọn thêm `Target Screen`. Nếu là `ShowSettings`, điền `Origin Screen`.
 5. Play thử scene, bấm từng nút để xác nhận flow đúng; muốn đổi đích thì chỉ cần chỉnh lại `Target Screen`.
