3. Bật Safe Area (nếu điện thoại có phần notch - camera trên đỉnh):
   
   Safe Area là gì? 
   → Trên một số điện thoại Android mới, phần camera ở trên đỉnh ăn mất không gian UI
   → "Safe Area" = "vùng an toàn" không bị che bởi camera
   → Nếu bật, UI tự động tránh vùng camera
   
   Cách làm:
   ├─ Click vào "Canvas" (Hierarchy)
   ├─ Ở Inspector (phía phải), tìm "Canvas Scaler" component
   ├─ Kéo xuống, tìm checkbox "Respect Safe Area"
   ├─ Tích ✓ vào (check box)
   └─ Done!
   
   Lưu scene:
   ├─ Ctrl + S

BƯỚC 1B: RESPONSIVE DESIGN PRINCIPLES (Để tất cả điện thoại vừa vặn)
═══════════════════════════════════════════════════════════════════════════════════════

🔑 KEY PRINCIPLES:

1️⃣ ANCHORS & PIVOT (Căn chỉnh linh hoạt)
   ────────────────────────────────────────

   ✓ Luôn set Anchor + Pivot TRƯỚC khi adjust Position/Size
   
   Common Anchor Presets:
   ├─ Full Screen:         Anchor: Stretch, Stretch
   ├─ Top Center:          Anchor: Top, Center X
   ├─ Bottom Center:       Anchor: Bottom, Center X
   ├─ Center:              Anchor: Center, Center
   ├─ Left Side:           Anchor: Left, Stretch (for sidebars)
   └─ Top Left:            Anchor: Top Left (for corners)

   Cách apply:
   → Rect Transform component → Anchor Presets (icon góc trái)
   → Hold ALT để preview, hold SHIFT để set pivot cùng lúc

2️⃣ LAYOUT GROUPS (Tự động sắp xếp components)
   ──────────────────────────────────────────

   VerticalLayoutGroup: Xếp con từ trên xuống
   ├─ Dùng cho: Menu items, level buttons, leaderboard list
   ├─ Setup:
   │  ├─ Add Component: VerticalLayoutGroup
   │  ├─ Spacing: 10 (khoảng cách giữa items)
   │  ├─ Child Controls: Width ✓, Height ✓
   │  ├─ Child Force Size: Width ✓, Height ✓
   │  └─ Preferred Size: Leave empty (auto calculate)
   │
   └─ Mỗi child cần Layout Element:
      ├─ Add Component: Layout Element
      ├─ Preferred Width: 400, Preferred Height: 80
      └─ Enable cả 2 ✓

   HorizontalLayoutGroup: Xếp con từ trái sang phải
   ├─ Dùng cho: 4 answer buttons, avatar selection
   ├─ Setup tương tự VerticalLayoutGroup
   ├─ Spacing: 15 giữa buttons
   ├─ Child Controls: Width ✓, Height ✓
   └─ Thường dùng kết hợp GridLayoutGroup

   GridLayoutGroup: Xếp thành lưới
   ├─ Dùng cho: Level selection 3x2, avatar grid 6 items
   ├─ Setup:
   │  ├─ Add Component: GridLayoutGroup
   │  ├─ Cell Size: 180 x 180 (adaptive)
   │  │  └─ Hoặc tính: Cell Width = (Screen Width - margins - spacing) / columns
   │  ├─ Spacing: 15 x 15
   │  ├─ Constraint: Flexible (tự wrap)
   │  └─ Child Layout Elements: Set preferred size
   │
   └─ Công thức responsive cho cell size:
      Cell Width = (Canvas Width - Left Padding - Right Padding - Spacing * (Columns - 1)) / Columns
      Ví dụ: (1080 - 40 - 40 - 15*2) / 3 = 320 each

3️⃣ IMAGE SCALING (Ảnh không bị méo)
   ────────────────────────────────

   ├─ Add Component: AspectRatioFitter (nếu ảnh fixed ratio)
   │  ├─ Aspect Ratio: 1 (square), 16:9, etc
   │  └─ Fit In Parent: ✓
   │
   └─ Image Component:
      ├─ Preserve Aspect: ✓
      └─ Layout Element: Set min/preferred size

4️⃣ FONT SCALING (Text vừa vặn tất cả màn hình)
   ────────────────────────────────────────

   KHÔNG hard-code font size!
   
   ├─ Use TextMeshPro với Canvas preferredSize
   ├─ Set Font Size = 48 (for 1080px canvas width)
   ├─ TextMeshPro sẽ auto-scale dựa Canvas Scaler
   │  └─ Hoặc dùng Dynamic mode + auto-shrink
   │
   └─ Layout Element:
      ├─ Preferred Width: -1 (auto)
      ├─ Preferred Height: -1 (auto)
      └─ Fit Preferred Size: ✓

5️⃣ SAFE AREA (Tránh notch Android)
   ───────────────────────────────

   ├─ Canvas → Canvas Scaler → Respect Safe Area: ✓
   │  (Tự động xử lý notch Android)
   │
   └─ Hoặc custom code (nếu cần debug):
      
public class SafeAreaDebugger : MonoBehaviour
      {
          private void Start()
          {
              Rect safeArea = Screen.safeArea;
              Debug.Log($"[SafeArea] Position: {safeArea.position}, Size: {safeArea.size}");
              Debug.Log($"[Screen] Width: {Screen.width}, Height: {Screen.height}");
              
              if (safeArea.width < Screen.width || safeArea.height < Screen.height)
                  Debug.Log("[SafeArea] NOTCH DETECTED - Canvas Scaler will handle it");
          }
      }

BƯỚC 2: TẠO CÁC PANEL (VỚI RESPONSIVE LAYOUT)
═══════════════════════════════════════════════════════════════════════════════════════

Trong GameUICanvas, tạo 12 Panels như sau:

[⭐ RESPONSIVE PATTERN ⭐]
Tất cả buttons, text fields phải dùng:
├─ Anchors: Top → Bottom (Stretch, Stretch) để fit screen
├─ Layout Groups: VerticalLayoutGroup để sắp xếp tự động

BƯỚC 3: TẠO UIManager.cs (Master Controller)
═══════════════════════════════════════════════════════════════════════════════════════

Tạo file: Assets/Script/Script_multiplayer/UIManager.cs

using UnityEngine;
using System.Collections.Generic;

public class UIManager : MonoBehaviour
{
    public static UIManager Instance { get; private set; }

    [SerializeField] private Dictionary<string, GameObject> panels = new Dictionary<string, GameObject>();
    
    // Panel references
    public GameObject welcomePanel;
    public GameObject loginPanel;
    public GameObject registerPanel;
    public GameObject mainMenuPanel;
    public GameObject modeSelectionPanel;
    public GameObject levelSelectionPanel;
    public GameObject difficultySelectionPanel;
    public GameObject inGamePanel;
    public GameObject pauseMenuPanel;
    public GameObject gameResultPanel;
    public GameObject leaderboardPanel;
    public GameObject profilePanel;
    public GameObject settingsPanel;

    private GameObject currentPanel = null;

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);
        InitializePanels();
    }

    private void InitializePanels()
    {
        // Register all panels
        panels.Add("Welcome", welcomePanel);
        panels.Add("Login", loginPanel);
        panels.Add("Register", registerPanel);
        panels.Add("MainMenu", mainMenuPanel);
        panels.Add("ModeSelection", modeSelectionPanel);
        panels.Add("LevelSelection", levelSelectionPanel);
        panels.Add("DifficultySelection", difficultySelectionPanel);
        panels.Add("InGame", inGamePanel);
        panels.Add("PauseMenu", pauseMenuPanel);
        panels.Add("GameResult", gameResultPanel);
        panels.Add("Leaderboard", leaderboardPanel);
        panels.Add("Profile", profilePanel);
        panels.Add("Settings", settingsPanel);

        // Hide all except welcome
        HideAllPanels();
        ShowPanel("Welcome");
    }

    public void ShowPanel(string panelName)
    {
        if (!panels.ContainsKey(panelName))
        {
            Debug.LogError($"Panel '{panelName}' not found!");
            return;
        }

        if (currentPanel != null)
            currentPanel.SetActive(false);

        currentPanel = panels[panelName];
        currentPanel.SetActive(true);
        Debug.Log($"[UI] Showing panel: {panelName}");
    }

    public void HideAllPanels()
    {
        foreach (var panel in panels.Values)
        {
            panel.SetActive(false);
        }
    }

    // Convenience methods for each panel
    public void ShowWelcomePanel() => ShowPanel("Welcome");
    public void ShowLoginPanel() => ShowPanel("Login");
    public void ShowRegisterPanel() => ShowPanel("Register");
    public void ShowMainMenuPanel() => ShowPanel("MainMenu");
    public void ShowModeSelectionPanel() => ShowPanel("ModeSelection");
    public void ShowLevelSelectionPanel() => ShowPanel("LevelSelection");
    public void ShowDifficultySelectionPanel() => ShowPanel("DifficultySelection");
    public void ShowInGamePanel() => ShowPanel("InGame");
    public void ShowPauseMenuPanel() => ShowPanel("PauseMenu");
    public void ShowGameResultPanel() => ShowPanel("GameResult");
    public void ShowLeaderboardPanel() => ShowPanel("Leaderboard");
    public void ShowProfilePanel() => ShowPanel("Profile");
    public void ShowSettingsPanel() => ShowPanel("Settings");
}

2. **Breakpoint-Based Design** (Android - tối ưu cho nhóm devices)
   
   public class AndroidResponsiveUIManager : MonoBehaviour
   {
       public enum ScreenSize { Small, Medium, Large }
       
       private ScreenSize currentSize = ScreenSize.Medium;
       
       private void Start()
       {
           DetermineScreenSize();
           ApplyLayout();
       }
       
       private void DetermineScreenSize()
       {
           float screenHeight = Screen.height;
           
           if (screenHeight < 1280)
               currentSize = ScreenSize.Small;      // Điện thoại nhỏ
           else if (screenHeight < 2000)
               currentSize = ScreenSize.Medium;     // Điện thoại vừa
           else
               currentSize = ScreenSize.Large;      // Điện thoại lớn / tablet
       }
       
       private void ApplyLayout()
       {
           switch (currentSize)
           {
               case ScreenSize.Small:
                   // Adjust: Button size nhỏ, Font size nhỏ
                   mainPlayButton.GetComponent<LayoutElement>().preferredHeight = 70;
                   break;
               case ScreenSize.Medium:
                   // Default layout
                   mainPlayButton.GetComponent<LayoutElement>().preferredHeight = 100;
                   break;
               case ScreenSize.Large:
                   // Adjust: Button size lớn, Font size lớn
                   mainPlayButton.GetComponent<LayoutElement>().preferredHeight = 120;
                   break;
           }
       }
   }
